Why multiprocessing spawn not fork: ONNX Runtime is not fork-safe — the session state gets corrupted when copied. Spawn starts a fresh Python interpreter per worker. Costs ~2 seconds once at the start of indexing, but runs correctly on all platforms including Windows.
Why batch size 8: A single CLIP image tensor is ~600KB. Eight of them = ~4.8MB, which fits in L3 CPU cache. At batch=32 you get 19MB per call — cache pressure slows things down. Batch=8 is the sweet spot on most laptops giving ~22ms per batch instead of 144ms for 8 sequential calls.
Why random port for the sidecar: If you hardcode 8000 and something else is using it, the sidecar fails silently and the app shows a blank screen. Random port selection finds a free port every time. The Rust shell reads it from the sidecar's stdout (SIDECAR_PORT=52341) and the React app asks Rust for it via invoke("get_sidecar_port").
Why one Python process instead of the Docker two-service split: On a desktop app, if the window closes everything dies together anyway — isolation between encoder and API buys you nothing. One process means one model load, no HTTP hop between encoder and search, and one simpler PyInstaller bundle.
Why onedir not onefile for PyInstaller: onefile packs everything into a self-extracting zip. On every launch it extracts to a temp folder — adds 3s startup, and Windows Defender frequently flags self-extracting executables as suspicious. onedir is a folder — Tauri bundles the whole folder, startup is instant.
The exact commands to run everything are in the README, step by step.


Why ONNX?

Good answer:

ONNX allowed me to run the CLIP model with INT8 quantization on CPU, reducing inference latency and removing the need for PyTorch dependencies.

Why FAISS IVFFlat?

Answer:

IVFFlat provides a good recall/latency tradeoff and allows tuning nprobe at query time without rebuilding the index.

Why multiprocessing instead of threads?

Answer:

Python GIL limits CPU-bound tasks, so multiprocessing allows parallel ONNX inference sessions.

Why Tauri instead of Electron?

Answer:

Tauri provides a much smaller binary (~10MB) and uses the system webview instead of bundling Chromium.
